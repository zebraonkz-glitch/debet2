# План разработки: «Результаты деятельности»

Документ описывает поэтапный план создания мобильного приложения на Expo / React Native для учёта доходов и расходов в разрезе проектов. Основан на [project.md](./project.md).

---

## 1. Обзор проекта

| Параметр | Значение |
|----------|----------|
| Название | Результаты деятельности |
| Платформа | iOS, Android (Expo Managed Workflow) |
| Архитектура | Вариант 1 — полностью мобильное приложение, без бэкенда |
| Язык | TypeScript |
| Хранение | Локально на устройстве (SQLite) |

### Ключевые возможности

1. Ввод доходов и расходов с привязкой к проекту.
2. Классификация расходов: прямые, постоянные (регулярные), долгоиграющие.
3. Правила распределения постоянных и долгоиграющих расходов по проектам.
4. Отчёт «Результаты деятельности» по проектам и периодам.
5. Экспорт отчёта в CSV.

### Вне scope первой версии (v1)

- Синхронизация с бэкендом (REST/GraphQL).
- Авторизация и мультипользовательский режим.
- Облачное резервное копирование.

## 3. Модель данных

### 3.1. Сущности

```
Project
  id, name, description?, isActive, createdAt

Category
  id, name, type: 'income' | 'expense_direct' | 'expense_recurring' | 'expense_long_term'

Transaction
  id, projectId?, categoryId, amount, date, comment?, createdAt
  — projectId обязателен для прямых операций и доходов

RecurringExpense (постоянный расход)
  id, categoryId, amount, period: 'monthly' | 'yearly', startDate, endDate?
  allocationRuleId

LongTermExpense (долгоиграющий расход)
  id, categoryId, totalAmount, startDate, endDate
  distributionMethod: 'linear' | 'manual'
  allocationRuleId

AllocationRule (правило распределения)
  id, name, method: 'proportional' | 'fixed_shares' | 'equal'
  — shares: JSON [{ projectId, share }]  (для fixed_shares)

ReportPeriod — вычисляемая сущность (не хранится)
```

### 3.2. Логика распределения расходов

| Тип расхода | Поведение |
|-------------|-----------|
| Прямой | 100% относится к выбранному проекту |
| Постоянный | Сумма за период распределяется по `AllocationRule` на все активные проекты |
| Долгоиграющий | `totalAmount` делится на месяцы (linear) или по ручным долям в рамках правила |

**Методы правила:**

- `proportional` — пропорционально доходам проектов за период (если доходов нет — поровну).
- `fixed_shares` — заданные доли в процентах (сумма = 100%).
- `equal` — поровну между выбранными проектами.

### 3.3. Расчёт отчёта за период

Для каждого проекта за `[dateFrom, dateTo]`:

```
доходы        = SUM(transactions WHERE type=income AND projectId)
прямые расходы = SUM(transactions WHERE type=direct AND projectId)
доля постоянных = allocate(recurring, project, period)
доля долгоиграющих = allocate(longTerm, project, period)
итого расходы = прямые + доля постоянных + доля долгоиграющих
результат     = доходы − итого расходы
```

### 4.2. Принципы UX

- Быстрый ввод: крупные кнопки «+ Доход» / «+ Расход», последние категории и проект по умолчанию.
- Минимум полей на одном экране; сложные настройки — отдельными шагами.
- Все суммы — с явным знаком и форматированием локали (`ru-RU`).
- Пустые состояния с подсказкой «Создайте первый проект».

## 8. Установка и запуск (целевые команды)

### Требования

- Node.js 18+
- Expo CLI / EAS CLI
- Android Studio или Xcode (либо EAS Build в облаке)

### Разработка

```powershell
# Клонирование и установка зависимостей
cd d:\20260707_work\debet2
npm install

# Запуск dev-сервера
npx expo start

# Запуск на Android-эмуляторе
npx expo start --android

# Запуск на iOS-симуляторе (macOS)
npx expo start --ios
```

### Сборка (EAS)

```powershell
# Установка EAS CLI (один раз)
npm install -g eas-cli

# Логин в Expo
eas login

# Конфигурация проекта
eas build:configure

# Сборка для Android
eas build --platform android --profile preview

# Сборка для iOS
eas build --platform ios --profile preview
```
